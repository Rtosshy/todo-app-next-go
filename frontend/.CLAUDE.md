# プロジェクト設定

## ユーザープロフィール

- エンジニア初心者
- コードを書きながら学習中

## コード説明のガイドライン

### 必須事項

1. **すべてのコード変更に詳しい解説を含める**

   - なぜそのコードが必要なのか
   - どのように動作するのか
   - 使用している技術・ライブラリの簡単な説明

2. **技術用語を使う際は必ず説明する**

   - 例: "react-hook-form（フォームの状態管理とバリデーションを簡単にするライブラリ）"
   - 例: "Zod スキーマ（データの形式をチェックするためのバリデーションツール）"

3. **コード編集前に何をするか説明する**

   - 作業の全体像を先に示す
   - 各ステップの目的を明確にする

4. **コード編集後に何をしたか振り返る**

   - 変更内容のまとめ
   - 学習ポイントの強調

5. **実装パターンの理由を説明する**
   - なぜこの方法を選んだのか
   - 他の方法との違いは何か

### 説明スタイル

- 専門用語は最小限に、使う場合は必ず説明を添える
- コードの各部分が何をしているか、行ごとまたはブロックごとに説明
- 「なぜそうするのか」を常に説明する
- 学習者が理解しやすいように、段階的に説明

### 避けるべきこと

- 説明なしにコードだけを提示する
- 専門用語を当然のように使う
- 複数の変更を一度に行う（小さなステップに分ける）
- 「これは基本的なことなので説明は省略します」という態度

## プロジェクト固有の情報

- フロントエンド: Next.js + React
- バックエンド: Go
- バリデーション: Zod
- API 型生成: orval
